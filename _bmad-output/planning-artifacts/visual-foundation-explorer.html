How It Works: One Attribute, Five Vibes

<!-- Host picks vibe at party creation -->
<body data-dj-state="ceremony" data-dj-vibe="kpop">

/* Dark surfaces stay constant across ALL vibes */
--dj-bg: #0a0a0f; /* always dark */
--dj-surface: #1a1a2e; /* always dark */
--dj-text: #f0f0f0; /* always readable */

/* Only ACCENTS change per vibe */
[data-dj-vibe="general"] { --dj-accent: #ffd700; }
[data-dj-vibe="kpop"]    { --dj-accent: #ff0080; }
[data-dj-vibe="rock"]     { --dj-accent: #ff4444; }
[data-dj-vibe="ballad"]  { --dj-accent: #ff9966; }
[data-dj-vibe="edm"]     { --dj-accent: #00ffc8; }
Implementation cost: ~20 lines of CSS custom property overrides. Zero JS logic. Zero new components.
The host taps one emoji when creating the party. That sets data-dj-vibe for the entire session.

Sally's Updated Recommendation

The vibe-adaptive system means you don't pick ONE color mood — the host picks the night's energy, and the palette follows. Here's what to decide:

  • Color: Vibe-Adaptive (all 5 vibes) — General as default, with K-pop/Rock/Ballad/EDM as host-selected options. Dark surfaces stay constant for readability. Only 4 accent tokens shift per vibe (~20 lines of CSS). Near-zero implementation cost.
  • Typography: A (Bold & Loud) with Space Grotesk — Constant across all vibes. Award titles SHOUT in any genre. Maximum dark-room readability. The font doesn't need to change per vibe — the color accent does all the genre-signaling work.
  • Density: B (Medium) — Constant across all vibes. Balanced for 3-hour sessions. Sparse is too empty, Dense is too risky for party conditions.
  • Bonus: Vibe-specific confetti + reactions — K-pop gets 💖💜, Rock gets 🔥⚡, Ballad gets ✨🌟, EDM gets 💎✨. Same sprite sheet system, just different emoji pool per vibe. Also near-zero cost.
  • Bonus: Vibe-flavored award copy — "Absolute showstopper" (General) vs "Main vocal energy" (K-pop) vs "Shredded the vocals" (Rock). Stored in constants/copy.js as vibe-keyed strings. One file, five flavor variations.
